# 🧠 LLM Model Requirements Calculator

**Микросервисная система для оценки совместимости LLM с вашим ПК**

## ✨ Возможности

- 🔍 **Автоматический сбор характеристик** – RAM, VRAM, модель GPU, ОС (Windows/macOS/Linux).
- 🧩 **Парсинг любых имён LLM** – расшифровка семейства, версии, размера (B), типа квантования, специализации (instruct/coder), MoE (например `8x7B`).
- 📊 **Расчёт требований к памяти** – веса модели + KV cache (с учётом контекста).
- 💡 **Рекомендации** – подойдёт ли модель для локального запуска, какой тип квантования выбрать, или стоит использовать облачный API.
- 🌐 **Микросервисная архитектура** – сервер (FastAPI) + кроссплатформенный CLI-клиент.
- 📈 **Расширяемость** – легко добавить поддержку новых семейств или методов квантования.

## ⚙️ Как это работает

1. Пользователь запускает **клиент** с названием модели (например `llama-3.3-70b-instruct-Q4_K_M`).
2. Клиент автоматически определяет:
   - Объём оперативной памяти (RAM)
   - Наличие и объём VRAM (видеопамяти)
   - Название GPU (NVIDIA/AMD/Intel/Apple Silicon)
   - Операционную систему и количество ядер CPU
3. Клиент отправляет JSON-запрос на **сервер**.
4. Сервер:
   - Парсит название модели → извлекает параметры (размер, квантование, специализацию)
   - Вычисляет необходимую VRAM для весов и KV cache (контекст 4096 токенов по умолчанию)
   - Сравнивает с возможностями ПК
   - Генерирует понятную рекомендацию (с эмодзи)
   - Возвращает JSON с деталями
5. Клиент выводит результат в консоль.

## 🧱 Архитектура

```
┌─────────────────┐      HTTP POST      ┌─────────────────┐
│   Клиент (PC)   │ ──────────────────► │   Сервер (API)  │
│  - сбор данных  │   /api/check_model  │  - парсинг      │
│  - отправка     │ ◄────────────────── │  - расчёт       │
└─────────────────┘      JSON ответ     │  - рекомендации │
                                        └─────────────────┘
```

- **Сервер**: Python + FastAPI + Uvicorn
- **Клиент**: Python + `psutil` + `requests` (без внешних зависимостей для GPU, используются встроенные утилиты ОС)

## 📦 Установка и запуск

### Требования
- Python 3.9+
- pip (менеджер пакетов)

### Сервер

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/llm-model-calculator.git
cd llm-model-calculator
```

2. Создайте виртуальное окружение (опционально):
```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows
```

3. Установите зависимости для сервера:
```bash
pip install fastapi uvicorn pydantic
```

4. Запустите сервер:
```bash
python server.py
```
Сервер будет доступен по адресу `http://localhost:8000`.  
Для публичного доступа укажите `--host 0.0.0.0`.

### Клиент

Клиент можно запускать на том же компьютере или на любом другом, указав адрес сервера.

1. Установите зависимости клиента:
```bash
pip install psutil requests
```
*Для точного определения VRAM на NVIDIA GPU можно установить `nvidia-ml-py3`, но это необязательно.*

2. Запустите клиент:
```bash
python client.py "название_модели"
```


### API вручную (через curl или Postman)

**Endpoint:** `POST /api/check_model`

**Пример запроса**:
```json
{
  "model_name": "mixtral-8x7b-instruct-v0.1-GPTQ",
  "system_info": {
    "ram_gb": 32.0,
    "vram_gb": 24.0,
    "has_gpu": true,
```

## 🧩 Как парсятся названия моделей

Поддерживаются следующие паттерны (примеры):

| Компонент        | Примеры                                           |
|------------------|---------------------------------------------------|
| Семейство        | `llama`, `qwen`, `mistral`, `gpt`, `claude`, `gemini` |
| Версия           | `3.3`, `2.5`, `v0.1`                              |
| Размер           | `7B`, `70B`, `405B`, `8x7B` (MoE)                 |
| Специализация    | `instruct`, `coder`, `chat`, `base`               |
| Квантование      | `FP16`, `Q8`, `Q4`, `GPTQ`, `GGUF`, `Q4_K_M`      |

**Алгоритм**:
- Регулярные выражения для поиска `число x число B` (MoE) или `число B`
- Поиск суффикса специализации и типа квантования
- Для MoE считается общее количество параметров (эксперты × размер эксперта), но активных экспертов принимаем 2 (как в Mixtral)
